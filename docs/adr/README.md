# ADR Index

Architecture Decision Records capture decisions that are important enough to constrain future implementation.

## Status Rules

- `Proposed` means the repo has a recommended direction but the user has not explicitly locked it in.
- `Accepted` means the direction should be treated as default unless a later ADR supersedes it.
- `Superseded` means keep for history only.

## Current ADRs

- `ADR-0001-esp-idf-first.md` - recommended primary firmware framework

## ADR Writing Rule

Promote a topic into an ADR only when it affects repository shape, framework choice, power strategy, provider interface shape, provisioning flow, or long-term maintenance cost.
